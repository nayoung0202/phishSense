# ADR-0003: 발송 설정당 허용 발신 도메인 수를 5개로 제한한다

- 상태: 승인
- 날짜: 2026-03-24

## 배경

발송 설정은 하나의 SMTP 연결 정보에 여러 허용 발신 도메인을 연결할 수 있습니다. 하지만 현재 제품 범위에서는 발신 도메인 검증, 테스트 발송, 프로젝트 생성 UX가 핵심이며, 별도 도메인 검증 상태 관리나 대규모 멀티브랜드 운영 기능은 아직 제공하지 않습니다.

이 상황에서 허용 발신 도메인 수를 무제한으로 열어두면 입력 UX와 운영 규칙이 모호해지고, 한 설정에 과도하게 많은 도메인을 몰아넣는 사용 패턴을 조기에 제어하기 어렵습니다.

## 결정

- 허용 발신 도메인은 발송 설정당 최대 5개까지만 등록합니다.
- 제한은 프론트엔드 입력 단계와 서버 저장 단계에 모두 동일하게 적용합니다.
- 개수 제한은 현재 단계에서 플랜별로 차등 적용하지 않습니다.

## 결과

- 대부분의 실제 사용 패턴인 1~3개 도메인 범위를 충분히 수용합니다.
- 설정 화면과 프로젝트 선택 UX가 과도하게 복잡해지지 않습니다.
- 향후 5개를 초과하는 요구가 반복되면, 단순 상향보다 별도 도메인 관리 기능 또는 검증 상태 모델 도입을 먼저 검토합니다.
